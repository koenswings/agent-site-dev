# Beacon Session Memory — 2026-04-01

## Atlas may commit to your repo

Atlas (operations-manager / COO) makes operational changes directly to coding agents' repos.
These are **not your work** — do not claim them or get confused by them.

**How to recognise them:**
- Commit message prefix: `ops:` or `memory:` authored by "Atlas" or "agent-operations-manager"
- They affect: `AGENTS.md`, `memory/*.md`, `outputs/*.md`
- They never touch source code, tests, or docs in your repo

**What to do:**
At every session start, after pulling main, run:
```bash
git log --oneline -10
```
If you see an `ops:` commit you don't remember, read the changed file to understand what Atlas
updated. It will be an AGENTS.md process change or a memory/output file.

**In practice:** Always run `git pull origin main` at session start. Your AGENTS.md may have
been updated by Atlas between sessions — reading it at startup (as your boot sequence requires)
will catch any changes automatically.
